# Live Helper Chat REST API Examples

This folder contains example scripts and utilities for interacting with the Live Helper Chat REST API. Below is a description of each file and its purpose.

---

## Core Files

### `lhrestapi.php`
**REST API Client Class** — The foundational class (`LHCRestAPI`) used by all example scripts. Handles HTTP requests to the LHC REST API using cURL with HTTP Basic Authentication. Supports GET, POST, PUT, and DELETE methods, URL parameters (unordered params), and JSON or XML response parsing.

**Dependencies:** None (standalone, requires cURL and PHP `json` extension)

**Usage:**
```php
$api = new LHCRestAPI($host, $username, $apiKey);
$response = $api->execute('chats', ['status' => 1], [], 'GET', true);
```

### `settings.ini.php`
**Configuration Template** — Returns an associative array with connection settings (`host`, `user`, `key`). Used by `live_example.php`, `live_workflow.php`, and other scripts to centralize API credentials.

## Example & Demonstration Scripts

### `example.php`
**Basic API Examples** — Demonstrates several common REST API calls with inline credentials:
- `setoperatorstatus` — Set operator online/offline
- `chats` — List chats with filters (status, department, user)
- `departaments` — List departments
- `fetchchat` — Fetch a single chat by ID
- `fetchchatmessages` — Fetch messages for a chat (supports `last_message_id` for pagination)
- XML response format example

### `live_example.php`
**Live API Examples (with settings.ini)** — Same operations as `example.php` but reads credentials from `settings.ini.php` and includes additional filter examples:
- `filtergt` — Greater-than filtering (e.g., `id > 1000`)
- `limit`/`offset` — Pagination
- `update_activity` — Force last activity update
- `setoperatorstatus` with `user_id`

### `live_example_offline.php`
**Visibility Status Example** — Demonstrates how to set an operator's visibility (online/offline) using `setvisibilitystatus`. Can identify the user by `user_id`, `username`, or `email`.

### `live_workflow.php`
**Complete Chat Workflow** — The most comprehensive example, illustrating a full visitor chat lifecycle:

1. **Initialization** — `chatcheckoperatormessage` with page view tracking (optional `vid`, `count_page`, visitor attributes)
2. **Start Chat** — `startchat` with visitor details (Email, Department, Username, Question, TOS, Phone). Supports `proactive` mode for invitation-based chats. Optionally links to a `vid`.
3. **Check Chat Status** — `checkchatstatus` polls until the chat is activated (operator assigned). Returns `activated`, `status`, `name_support`, `closed` flags.
4. **Fetch Messages** — `fetchchatmessages` retrieves new messages with `last_message_id` pagination, `ignore_system_messages`, and `workflow` options.
5. **Send Message** — `addmsguser` posts a message as the visitor.
6. **Close Chat** — `closechatasvisitor` closes the chat from the visitor side (supports `eclose` flag for explicit close).
7. **Update Attributes** — `updatechatattributes` modifies chat custom attributes.

### `merge_vid.php`
**Visitor ID Merge** — Shows how to merge two visitor IDs (VIDs) using the `setnewvid` endpoint. Useful when a returning visitor needs to be re-identified under a different VID.

### `webhook.php`
**Incoming Webhook Example** — Demonstrates sending a JSON payload to an incoming webhook endpoint (`/webhooks/incoming/{identifier}`). Shows the expected payload format for chat messages including `chatId`, `computer`, `chatName`, `type`, `date`, `body`, and `sender` fields.

## Data Export & Utilities

### `chats_export.php`
**Bulk Chat Export Script** — Incrementally exports chats using the `restapi/chats` endpoint with `id_gt` pagination. Key features:
- **JSONL output** — One JSON object per line
- **Resumable** — Tracks progress in `chats_export_progress.json` so exports can be interrupted and resumed
- **Configurable delay** — Skips recent chats (default 30 min) to avoid exporting in-progress conversations
- **Prefill fields** — Supports `prefill_fields` for enriched data (e.g., `messages_statistic`, `subject`, `department_name`)
- **Sorting** — Exports in ascending ID order (`id_asc`)

**Usage:** `php chats_export.php`

### `chats_export_progress.json`
**Progress Tracking File** — Created and updated by `chats_export.php`. Stores `last_id` (last exported chat ID), `total_exported` (count), and `exported_at` (timestamp). Enables resume capability.

### `chats_export.jsonl`
**Export Output Format** — JSON Lines file format (`.jsonl`) where each line is a JSON object representing one exported chat. This is the output file generated by `chats_export.php`.

### `conversations_export.php`
**Bulk Mail Conversations Export Script** — Incrementally exports mail conversations using the `restapi/conversations` endpoint with `id_gt` pagination. Key features:
- **JSONL output** — One JSON object per line
- **Resumable** — Tracks progress in `conversations_export_progress.json` so exports can be interrupted and resumed
- **Configurable delay** — Skips recent conversations via `timetots` (default 30 min)
- **Prefill fields** — Supports `prefill_fields` (e.g., `mailbox`, `mail_variables_array`)
- **Include messages** — Optionally embeds all messages within each conversation via `include_messages`
- **Sorting** — Exports in ascending ID order (`id_asc`)

**Usage:** `php conversations_export.php`

### `conversations_export_progress.json`
**Progress Tracking File** — Created and updated by `conversations_export.php`. Stores `last_id`, `total_exported`, and `exported_at`. Enables resume capability.

### `conversations_export.jsonl`
**Export Output Format** — JSON Lines file (`.jsonl`) where each line is a JSON object representing one exported mail conversation.

### `conversations_messages_export.php`
**Bulk Mail Messages Export Script** — Incrementally exports individual mail messages using the `restapi/conversationsmessages` endpoint with `id_gt` pagination. Key features:
- **JSONL output** — One JSON object per line (message-level, not conversation-level)
- **Resumable** — Tracks progress in `conversations_messages_export_progress.json`
- **Configurable delay** — Skips recent messages via `timetots` (default 30 min)
- **Prefill fields** — Supports `prefill_fields` (e.g., `conversation`, `department`, `mailbox`, `user`, `plain_user_name`, `files`, `attachments`, `subjects`, `body_front`, `wait_time_pending`, `wait_time_response`, etc.)
- **Include conversations** — Optionally embed parent conversation objects via `include_conversations`
- **Sorting** — Exports in ascending ID order (`id_asc`)

**Usage:** `php conversations_messages_export.php`

### `conversations_messages_export_progress.json`
**Progress Tracking File** — Created and updated by `conversations_messages_export.php`. Stores `last_id`, `total_exported`, and `exported_at`.

### `conversations_messages_export.jsonl`
**Export Output Format** — JSON Lines file (`.jsonl`) where each line is a JSON object representing one exported mail message.
